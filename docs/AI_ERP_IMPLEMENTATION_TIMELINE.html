1. Executive Summary

This timeline lays out the complete 38-week build of an AI-driven ERP and CRM platform, delivered in four phases. Each phase produces a working, demonstrable agent that can be put in front of real users before the next phase begins — a deliberately incremental approach that derisks the program and lets the business start capturing value from Week 12 onward.

Phase Focus Weeks First user value Headline deliverable
Phase 1 AI Employee Agent (HR) 1 – 12 Self-service leave, attendance, payslip Conversational HR assistant in production
Phase 2 AI CRM Agent (Sales) 13 – 20 Faster lead-to-quote cycle Lead, quote, follow-up agent
Phase 3 AI Operations Agent 21 – 28 Touchless procurement / inventory Procurement & inventory agent
Phase 4 Full Agentic ERP 29 – 38 Cross-module workflows, voice everywhere Unified AI-first enterprise OS
Delivery Philosophy
Each phase ships a usable agent. We do not wait until Week 38 to launch. By Week 12 employees are already using the HR agent in production while the CRM agent is being built in parallel.

4. Phase 1 — AI Employee Agent (Weeks 1–12)

Phase 1
AI Employee Agent — HR automation
Duration: 12 weeks (~3 months) · Outcome: production HR assistant covering leave, attendance, payroll, onboarding

Weeks 1–2 — Foundation

  • Install Pydantic AI in the existing FastAPI backend
  • Set up app/agents/ folder + base Agent dependency injection container
  • Define the first five tools as Python wrappers over existing routes: get_employee, get_leave_balance, apply_leave, get_attendance, get_payslip
  • Add /agent/chat endpoint with the base agent (single-agent, no routing yet)
  • Audit-log table extended with triggered_by_agent flag
  • Confirmation pattern: every state-changing tool returns a draft, requires explicit user "yes"

Weeks 3–4 — Leave Agent (production)

  • Build the leave-request agent on top of the existing leave module (already started)
  • Intent classification + entity extraction via Gemini structured output
  • Balance/quota validation against existing leave_balance table
  • Confirmation prompt with full summary
  • Submit creates an existing leave_request row; manager email approval flow remains unchanged
  • Employee Portal Leave tab: chat widget + live status panel
Milestone M1 (Week 4)
First production agent. 5 employees use it daily for a week without intervention.

Weeks 5–6 — Attendance + Payroll Q&A

  • Read-only agents (cannot modify attendance/payroll — safer first launch)
  • Sample queries: "How many late marks this month?", "Send me my last 3 payslips", "What's my net pay this month?"
  • Payslip download tool: agent calls existing PDF generator
  • Permission-scoping: employee sees only own records; HR sees team

Weeks 7–9 — Onboarding Agent

  • Conversational onboarding for HR: "Create profile for Aniket, joins 1 July, Sales, reports to Ram"
  • Document collection step: Aadhaar / PAN / Resume upload through chat
  • OCR extraction stub (full OCR comes in Phase 4)
  • Cross-checks: duplicate phone/email, manager existence, department validation
  • Auto-creates employee + sends welcome email through existing email service

Weeks 10–11 — RAG memory + voice

  • Add Chroma as embedded vector store
  • Backfill embeddings for: employees, HR policies, leave rules, holiday calendar
  • Retrieval tool recall_about(topic) available to all Phase 1 agents
  • Voice input via Web Speech API (Indian English, Tamil, Hindi)
  • Text-to-speech for agent replies (browser SpeechSynthesis)

Week 12 — Hardening & Phase 1 launch

  • Per-user daily token budget + cost alarms
  • Rate limiting at FastAPI middleware
  • Hallucination guardrails: all numerical facts (balance, hours, salary) rendered from DB, never from LLM
  • End-to-end UAT with full HR team (5 days)
  • Production cut-over; legacy form-based screens remain as fallback for 30 days
Milestone M2 (Week 12)
Phase 1 GA. Entire employee population can self-serve via chat or voice. HR ticket volume target: down 60%.

5. Phase 2 — AI CRM Agent (Weeks 13–20)

Phase 2
AI CRM Agent — Sales automation
Duration: 8 weeks (~2 months) · Outcome: lead-to-quote cycle compressed from days to minutes

Weeks 13–14 — Lead Agent

  • "Create a lead from this conversation" — agent extracts contact info, qualifies BANT, schedules follow-up
  • Bulk lead import via natural language ("Add these 30 contacts from yesterday's expo")
  • Customer 360 mini-view: agent surfaces full history when asked about a known account
  • Tools: create_lead, find_customer, schedule_follow_up, log_interaction

Weeks 15–16 — Quotation Generation Agent

  • "Quote Apollo Hospitals for 5 units of C 608 R1 with 10% margin" — agent looks up BOM, applies pricing rules, generates quotation
  • Multi-line item conversational entry: "add 3 more of model X"
  • Approval routing: above ₹5L auto-routes to MD via existing email flow
  • PDF generation reuses existing quotation template + company master settings

Weeks 17–18 — Follow-up & Pipeline Intelligence

  • Daily morning brief: "3 leads going stale, 2 quotes expiring tomorrow, follow-ups due today"
  • Smart suggestions: "Apollo hasn't responded in 14 days — want to send a follow-up?"
  • Sentiment tracking: tags conversations as warm / cold / negotiating based on last interaction
  • RAG over CRM history: "What did we discuss with Apollo last quarter?"

Weeks 19–20 — UAT & rollout

  • Sales team UAT (2 weeks, 5 reps)
  • Tune prompts for sales language (industry jargon, abbreviations)
  • Pipeline accuracy spot-checks vs Salesforce-like baselines
  • Phase 2 GA with full sales team
Milestone M3 (Week 20)
Phase 2 GA. Average lead-to-quote time target: under 15 minutes (vs ~4 hours baseline).

6. Phase 3 — AI Operations Agent (Weeks 21–28)

Phase 3
AI Operations Agent — Procurement & inventory
Duration: 8 weeks (~2 months) · Outcome: touchless purchase & stock workflows

Weeks 21–23 — Procurement Agent

  • "Raise a PO for 200 units of SMPS 24V 10A from Steel Mart" — agent drafts PO, checks vendor master, routes for approval
  • Bulk RFQ generation: "Send RFQ for these 15 items to our 3 preferred suppliers"
  • Three-way matching support: PO vs GRN vs Invoice flagged on mismatch
  • Vendor performance summaries on demand

Weeks 24–25 — Inventory Agent

  • "What's running low?" — agent lists items below reorder threshold
  • "Move 50 units of glass door from Coimbatore to Chennai" — auto-creates transfer note
  • Cycle-count guidance: "Which 20 SKUs should we count this week based on movement?"
  • Stock-aging insights for finance

Weeks 26–27 — Vendor Management & Approvals

  • Vendor onboarding conversational flow (similar to employee onboarding)
  • Multi-level approval router: PR → PO → payment with policy-driven escalation
  • Spend analytics on demand: "How much did we spend on electronics this quarter?"

Week 28 — Phase 3 UAT + launch

Milestone M4 (Week 28)
Phase 3 GA. Touchless PO & transfer workflows live across procurement & warehouse.

7. Phase 4 — Full Agentic ERP (Weeks 29–38)

Phase 4
Full Agentic ERP — cross-module orchestration
Duration: 10 weeks (~2.5 months) · Outcome: unified AI-first enterprise OS

Weeks 29–31 — Cross-module orchestration

  • Single global AI command palette (Ctrl+K) on every screen
  • Intent router: classifies each message into command vs query vs navigate vs answer, then dispatches to the right sub-agent
  • Multi-agent flows: "Apollo wants a quote and a 30-day credit line" — CRM agent + finance agent collaborate
  • Customer 360 unified view (already designed) — agent answers any question about any customer

Weeks 32–33 — Predictive analytics

  • Daily executive brief generated by AI at 8 AM: revenue, pipeline, inventory alerts, employee alerts
  • Anomaly detection: "Spending on raw materials up 22% MoM — want a breakdown?"
  • Cash-flow forecast based on PO + invoice + payment data
  • Customer churn-risk flagging

Weeks 34–35 — Voice & local LLM fallback

  • Install Ollama with Llama 3.1 8B on a dedicated GPU box or office workstation
  • Classifier rule: sensitive queries (salary, bank details, KYC) auto-route to local model; rest goes to Gemini
  • Voice everywhere: every chat surface accepts voice input + reads back replies
  • Multilingual: Tamil + Hindi + English Indian + Tanglish all natively supported

Weeks 36–37 — OCR & document understanding

  • Self-hosted OCR (Tesseract) for Aadhaar, PAN, GST certificates, invoices
  • Document upload → auto-classify → auto-extract fields → pre-fill the next agent action
  • Invoice three-way match becomes single-conversation operation

Week 38 — Stabilisation & production GA

  • Cloudflare named tunnel (erp.bvc24.com) replaces quick tunnel
  • Backup & disaster recovery rehearsed
  • Compliance review (DPDP Act audit)
  • Documentation: user guide, admin guide, API reference
  • Full production launch
Milestone M5 (Week 38)
Phase 4 GA. AI-first ERP fully live. Employees, managers, sales, procurement & finance all use natural-language interfaces by default.

10. Risk Register

Risk Likelihood Impact Mitigation
LLM provider pricing or T&Cs change MediumMedium Pydantic AI is provider-agnostic; swap to Claude / Ollama in one config line. Local fallback ready by Week 34.
Hallucinated facts cause wrong actions High (initial)High All numerical/identifying data rendered from DB, never from LLM. Tool-call required for any state change. Confirmation prompt before every write.
Users distrust the AI HighMedium Confirmation step + complete audit trail visible to user. Legacy forms kept as fallback for 30 days after each Phase GA.
Internal PII leaks via cloud LLM logs MediumHigh Strip PII from prompts; route sensitive queries to local Ollama after Phase 4 Week 34.
Vendor outage (Gemini / Anthropic / OpenAI) Low-MedMedium Provider chain in Pydantic AI: Flash → Pro → Ollama → graceful "try again" message.
Cost spirals with adoption MediumMedium Per-user daily token budget + cost alerts at 80%. Hard cap at 100% with friendly error.
Single-developer dependency HighHigh Documented architecture, simple frameworks, small surface area. Onboard a second engineer by Phase 3.
Scope creep extending timeline HighHigh Strict phase gates. New asks parked for the next phase, never inserted mid-phase.
Voice STT accuracy poor for regional accents MediumLow Web Speech API tested in Phase 1; if quality insufficient, fall back to text-only and revisit with Whisper / Google STT in Phase 4.

13. Change-Management Plan

Communication

  • Week -1 of each phase: town-hall preview to affected users
  • Weekly status email to leadership
  • Anonymous feedback form open throughout the program

Training

  • Phase 1 launch: 30-min hands-on session for all employees
  • Phase 2 launch: deep dive for sales (90 min)
  • Phase 3 launch: workshops for procurement & warehouse
  • Phase 4: short YouTube-style explainer videos (3–5 min each) for self-service learning

Fallback & Coexistence

  • Existing form-based screens remain accessible throughout the program
  • Each phase has a 30-day "shadow mode" where the AI and the form both work, then AI becomes primary
  • Power users always retain the option to use the legacy forms for edge cases

Adoption metrics monitored weekly

  • Daily active agent users / total employees
  • Tasks completed via chat vs forms
  • User satisfaction (in-product thumbs up / down on every reply)
  • Error rate & "I gave up" rate (conversations abandoned without completion)
Final Note
The timeline is deliberately aggressive but realistic for a focused single-engineer program with strong SME support. Each phase gate is a decision point — the program can pause, accelerate, or re-scope at any of the five milestones without leaving the business in an unusable state.
