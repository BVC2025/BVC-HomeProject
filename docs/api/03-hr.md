## Employees

| Method | Path | Purpose |
|---|---|---|
| POST | `/create-employee` | Admin creates employee |
| GET | `/employees` | Directory list (optional `?status=` `?department_id=`) |
| GET | `/employees/{id}` | Detail with resolved dept/designation/role |
| GET | `/employees/by-code/{code}` | Lookup by `EMPLOYEE_CODE` |
| POST | `/employees/by-code/{code}/submit-profile` | Self-registration submission |
| PUT | `/update-employee/{id}` | Update record |
| PUT | `/employees/{id}/reset-password` | Admin password reset `{ NEW_PASSWORD }` |
| POST | `/employees/{id}/upload-photo` | Multipart photo upload |
| DELETE | `/delete-employee/{id}` | Remove |
| POST | `/employees/wipe-all` | Dev â€” clear all employees (gated) |

## Attendance

| Method | Path | Purpose |
|---|---|---|
| POST | `/attendance/check-in` | Manual check-in `{ EMPLOYEE_ID }` |
| POST | `/attendance/check-out` | Manual check-out |
| POST | `/attendance/mark-absent` | Mark absent `{ EMPLOYEE_ID, DATE, REASON }` |
| GET | `/attendance` | List with filters `?date=YYYY-MM-DD&employee_id=&status=` |
| GET | `/attendance/today` | Today's records |
| GET | `/attendance/live-board` | Real-time active check-ins |
| DELETE | `/attendance/{id}` | Remove |
| GET | `/employee-attendance` | Aggregated summary |

## Biometric

| Method | Path | Purpose |

## Leave

| Method | Path | Purpose |
|---|---|---|
| POST | `/leave/apply` | Apply for leave `{ EMPLOYEE_ID, LEAVE_TYPE, START_DATE, END_DATE, REASON }` |
| GET | `/leave/decide/{token}?action=approve|reject` | Email link decision (renders HTML) |
| PATCH | `/leave/{id}/approve` | Admin approve |
| PATCH | `/leave/{id}/reject` | Admin reject `{ REJECTION_REASON }` |
| GET | `/leave/pending` | Pending approvals |
| GET | `/leave/all` | All requests (filterable) |
| GET | `/leave/my-requests` | Current employee's history |
| PATCH | `/leave/{id}/cancel` | Cancel own request |
| GET | `/leave/balance/{employee_id}` | Annual quota remaining |
| GET | `/leave/dashboard` | Leave analytics |
